Prototype overview

  • A user can create an account.
  • Their email address is verified.
  • The verify email page automatically updates when the user selects link in email.
  • The user can create their password and their account is created.
  • The user does not need to sign in (as in Version 1) they can now view their account directly after creating their account.
  • Once in the account they can manage their companies and add authorised people within the account (roles have been removed).

Adding a company - Explored three scenarios for authentication code:

  • (1) Yes – The user has the authentication code
  • (2) No – The user does not have the authentication code and there are no authorised people for the company (online)
  • (3) No - User does not have the authentication but can request to be authorised to file for the company
  • Once a company is added to the account. They can then file for this company.

Email journey - Request to add you as an authorised person for FLOWERS LIMITED

  • Email request to become an authorised person
  • Confirm or deny request
